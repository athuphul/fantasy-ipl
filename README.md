# Fantasy IPL 2026

A live fantasy cricket tracker for IPL 2026. Ten friends drafted squads before the season — this site scores every match automatically and maintains a season-long leaderboard.

**Live site:** [atharvap.me/fantasy-ipl](http://atharvap.me/fantasy-ipl/)

## How it works

Each manager picked 11-14 IPL players before the season started. Rosters are locked — no transfers. After every match, a scoring script runs automatically, computes fantasy points for every player, and publishes updated standings to GitHub Pages.

Only each team's **top 11 cumulative scorers** count toward the final standings. Captains earn **2x** points and vice-captains **1.5x**.

### Scoring rules

| Category | Action | Points |
|----------|--------|--------|
| **Batting** | Per run | +1 |
| | Boundary (4) | +1 bonus |
| | Six (6) | +2 bonus |
| | Half-century | +10 |
| | Century | +30 |
| | SR >= 200 (min 10 balls) | +15 |
| | SR < 75 (min 10 balls) | -10 |
| | Duck (dismissed for 0) | -5 |
| **Bowling** | Wicket | +25 |
| | 3W / 4W / 5W haul | +10 / +20 / +30 |
| | Economy <= 6.0 (min 2 ov) | +20 |
| | Economy >= 14.0 (min 2 ov) | -15 |
| **Fielding** | Catch | +8 |
| | Stumping | +10 |
| | Direct run out | +10 |
| | Run out assist | +5 |

Full rules with all tiers are on the [Scoring tab](http://atharvap.me/fantasy-ipl/) of the site.

## Architecture

```
index.html + app.js + style.css    Frontend (static, no build step)
          |
          reads
          |
    data/scores.json               Computed every ~10 min by CI
          |
    generated by
          |
  scripts/update-scores-espn.js    Scoring engine (Node.js)
          |
    fetches from 3 sources
          |
    +-----------+-----------+
    |           |           |
  ESPN      CricAPI     Cricbuzz
 (points)  (run outs)  (scorecard)
```

### Data sources

| Source | Method | Auth | What it provides |
|--------|--------|------|------------------|
| **ESPN Cricinfo** | REST API | None | Live player stats (runs, wickets, catches, economy, etc.) — primary source for computing fantasy points |
| **CricAPI** | REST API | API key | Run out fielder names (ESPN doesn't include these) |
| **Cricbuzz** | HTML scraping | None | Authoritative scorecard with correct dismissal text — used for fielding point derivation and display |

### Why three sources?

ESPN is fast and free but has two problems: (1) `stats.caught` is sometimes wrong, and (2) it doesn't include run out fielder names. CricAPI has run out data but is rate-limited. Cricbuzz has the most accurate dismissal text ("c Ishan Kishan b Tyagi"), which we parse to derive correct fielding points — catches, stumpings, and run outs.

The scoring engine uses ESPN for batting/bowling point computation, then **overrides** ESPN fielding with Cricbuzz-derived fielding by parsing every dismissal string in the scorecard.

### Pipeline per run

1. **Reprocess incomplete matches** — any match with `isComplete: false` gets re-fetched and re-scored
2. **Check schedule window** — exit early if no match is live or recently completed
3. **Fetch ESPN header** — discover which IPL matches are pre/in/post
4. **Discover Cricbuzz IDs** — scrape the Cricbuzz series fixtures page to map ESPN event IDs to Cricbuzz match IDs for any new matches
5. **For each live/completed match:**
   - Fetch ESPN summary → compute batting, bowling, fielding points
   - Fetch CricAPI scorecard → supplement run out fielding credits
   - Fetch Cricbuzz scorecard → parse dismissal text, override fielding points
   - If match complete: cache to `data/fantasy_scores/` and `data/match_scores/`
6. **Build leaderboard** — sum all matches, apply captain/VC multipliers, rank top-11 scores
7. **Commit and push** `data/scores.json` → GitHub Pages auto-deploys

### Caching

Completed matches are cached to `data/fantasy_scores/{matchId}.json`. On subsequent runs, cached matches are skipped entirely — no API calls. Only live and incomplete matches are re-fetched.

## CI/CD

| System | Trigger | Purpose |
|--------|---------|---------|
| **CircleCI** | Cron (every ~10 min during matches) | Runs `node scripts/update-scores-espn.js`, commits and pushes score updates |
| **GitHub Actions** (`deploy-pages.yml`) | Push to `main` | Deploys entire repo to GitHub Pages |
| **GitHub Actions** (`update-scores.yml`) | Manual dispatch | Backup trigger for score updates |

Score update commits include `[skip ci]` to avoid re-triggering CircleCI.

## Frontend

Static HTML/CSS/JS — no frameworks, no build tools. Opens `index.html` directly in a browser.

- **Leaderboard** — rankings with form bars, last-match score, gap to leader
- **Matches** — expandable cards with full scorecard (batting + bowling + fielding), fantasy point annotations, and match-day winner
- **Players** — sortable table of all fantasy players across teams; hero tiles for best batsman, bowler, and all-rounder
- **Scoring** — full rules reference
- **Dark/light theme** — toggle in header, persisted to localStorage

Data refreshes on page load from `data/scores.json`. During live matches, the current match card shows real-time scores and fantasy points.

## Local development

No dependencies to install for the frontend — just open `index.html` in a browser.

To run the scoring script locally:

```bash
# Optional: set CricAPI keys for run-out data
export API_KEY_1=your_key_here

# Run the scoring engine
node scripts/update-scores-espn.js
```

The script will fetch live data, compute scores, and write to `data/scores.json`. Refresh the browser to see updates.
